Home Page

Technical Description

  • <header> - Contains the navigation bar with links to different pages like "Team," "Sitemap," "Profile," etc. The website's logo is also placed here.
  • <nav> - Holds the navigation links inside the header, allowing users to move between different sections of the website.
  • <main> - Contains the main content of the homepage, including success statistics, mission, vision, and content tiles.
  • <section> - Used to divide the page into different sections, such as Mission, Vision, and 90% Success! Each section has a heading and descriptive text.
  • <div> - Used multiple times for layout and styling, such as wrapping images, aligning text, and creating a structured design.
  • <h2> - Defines headings for different sections like "Our Mission," "Our Vision," and "90% Success!" to highlight important content.
  • <footer> - Contains copyright information and a link to the page editor, ensuring that users can access footer details at the bottom of the page.

Accessibility

For the homepage, I focused on ensuring accessibility through a few key design and functionality choices. To improve readability for users with visual impairments, I used high contrast colors between the text and background. The navigation bar is structured with the <nav> tag, which aids in clear navigation for both screen readers and keyboard users. The sticky navigation ensures that users can always easily access important links no matter where they are on the page.
I also used the <section> tag to divide the content into distinct parts such as the mission, vision, and success statistics, making it easier for screen readers to identify and navigate between different sections. The use of <div> tags helps with layout and positioning without cluttering the page, making it more digestible.
For readability, I chose a clean, simple font and ensured good line spacing, which benefits users with dyslexia and other cognitive challenges. These design decisions enhance the user experience, making the website accessible to a wider audience.

Challenges and Lessons Learned

Building the website came with its challenges, especially ensuring it was responsive across devices and accessible to all users. The navigation bar needed to be easy to use for both screen readers and keyboard users, which led to using semantic HTML tags like <nav> and <section>.
The table page required careful design to ensure data was displayed clearly and was accessible. Testing across devices taught me the importance of making sure everything works smoothly, no matter the screen size.
Overall, I learned the value of simple, clear design and accessibility, making the website usable for a wider audience, including those with visual impairments or cognitive challenges.
