Feedback Page

Technical Description

The Feedback Page is implemented using HTML, CSS, and JavaScript to ensure both functionality and a visually appealing design.

Key Features:

  • Form Elements: Includes text inputs, radio buttons, file upload, and a star rating system.
  • Pagination: Splits feedback collection into steps for better usability.
  • Interactive Features: Star ratings, range inputs, and a scroll-to-top button.
  • Navigation & Footer: Clear site navigation and copyright info.

Styling (CSS):

  • Modern Design: Uses border-radius, box-shadow, and a dark theme with blue highlights.
  • Responsive & Readable: Ensures mobile adaptability and uses the ‘Poppins’ font.
  • Dynamic Effects: Hover transitions and interactive elements enhance UX.

Accessibility

  • Semantic HTML & ARIA Support: Enhances screen reader compatibility.
  • Keyboard Navigation: All elements are accessible via keyboard.
  • Form Validation: Inline error messages for user guidance.

Team Page

Technical Description

HTML & CSS Techniques

  • HTML Structure: Uses [header], [nav], [main], and [footer] for a well-organized layout.
  • Grid Layout: display: grid; with grid-template-columns: repeat(auto-fit, minmax(500px, 1fr)); ensures responsiveness.
  • Card Styling: Dark backgrounds, border-radius, and box-shadow for a polished look.
  • Hover Effects: opacity: 0.3; fades images, while .extraContent appears on hover.
  • Smooth Transitions: transition: opacity 0.3s ease; for interactive elements.

Accessibility

  • Semantic HTML: Improves screen reader support.
  • Alt Text for Images: Enhances accessibility for visually impaired users.
  • High Contrast & Readability: Dark background with white text for clarity.
  • Keyboard Navigation: Logical navigation order ensures usability.
  • Responsive Design: Adjusts to different screen sizes dynamically.

Content Page

Technical Description

This content page is implemented using a combination of HTML and CSS techniques to achieve an engaging, structured, and accessible web design. Below is an explanation of the key elements used.

HTML Structure

  • [!DOCTYPE html] ensures modern browser compatibility.
  • [html lang="en"] sets the language for accessibility.
  • Navigation (nav) and Sidebar (sidebar) provide easy navigation.
  • Content (container) and Cards (card) display the main content.
  • Footer (footer) includes copyright information.

CSS Styling

  • Global styling sets consistent fonts and background.
  • Navigation uses flexbox for alignment with smooth hover effects.
  • Cards have a modern, clean look with spacing and shadows.
  • Sidebar is sticky and scrollable.
  • Mobile Design adapts layout for smaller screens.

Accessibility

  • Semantic HTML: Uses proper HTML tags for better structure and accessibility.
  • Color and Contrast: High-contrast text for easier reading.
  • Keyboard Navigation: Links are accessible using a keyboard with visual focus indicators.
  • ARIA: Adds extra features for screen readers, like role="navigation" for the [nav] element.

Responsive Design

  • The layout adjusts automatically for different screen sizes using flexbox and media queries.

Challenges and Lessons Learned

During implementation, challenges like code errors, style conflicts, JavaScript issues, image path problems, page connection failures, form creation, rating functionality, character counting, and error handling arose. These were resolved through debugging, consistent file organization, and proper event linking. A key lesson was avoiding variable reuse for color identification to prevent style conflicts. Smooth transitions for moving cards were achieved using `transform`, and form validation and error handling were improved by ensuring proper input checks. Overall, the process emphasized the importance of attention to detail, organization, and testing.

Compliance

To ensure compliance with Janet regulations, the 'Life Below Water' website prioritizes accessibility, security, and good design practices. It includes features like alt text for images to aid screen readers, uses HTTPS for secure browsing, and ensures mobile responsiveness for a smooth user experience. The website also maintains clear and intuitive navigation, making it easy for users to find information. These steps help make the site accessible to all users while keeping it secure and user-friendly, adhering to the key guidelines set by Janet for web page publication.
